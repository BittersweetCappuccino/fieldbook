User Flows for the Shop Floor

Fieldbook routes a piece of equipment through five departments the way a dealership once routed a paper work-order ticket. Every flow below is a real path through the mockup — nodes are screens that exist, arrows are the action that advances the record, and each lane's color is the department's wayfinding color, exactly as it appears in the app.

How this map was drawn

Every node and arrow is traced from the mockup itself — the action router in js/sidebar.js, the workflow screens in js/workflow.js, and the shared entity metadata that maps each record to its department, detail page, and Actions menu. Palette and type reuse css/styles.css, so the flow board reads in the same shop-floor language as the app.
